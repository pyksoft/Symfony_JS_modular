Getting started

How to use and setup Gear, compatibility and stuff.

Quick setup

1/ Download

gear comes in two versions. A production version to use as-is, and a development version, containing all source files. If you plan to modify the theme (which is our goal !) we recommend to use the development version of Gear.

gear - production

For a simple use of Gear, just download the precompiled version (min). You can use the minified version (gear.min.css) or the full css.

Download prodution gear
gear - development

Contains the Sass files ; choose this version if you want to create a unique theme and take control of gear ! A default theme (scss) is included.

Download development gear

2/ Installation

Production / precompiled version

Just drop the 'gear' folder content in your project ; you can use the css classes right away ;

Or use one of our template : -examples-
That's it !

Development version

Open the .scss files in your favorite editor and start editing parameters following the documentation. The files are located in the 'gear/scss' folder. Then you can compile your .scss into working css, and edit your html. No big deal !

3/ What's inside ?

gear - production

gear/
├── dist/
│   ├── gear.css
│   └── gear.min.css
│
└── js/
    ├── gear.js
    └── gear.min.js

You will find the css, both minified and compiled, as well as the default theme (again minified and full versions). This is the basic Gear, you can use it to quickly prototype or produce your website. If you want to have more control switch to the production version !

You will also need a Sass compiler to save and use your changes. We recommand using Prepros or Grunt.js. Following the docs you will be able to tweak Gear to suit your needs and your design.

4/ Things you can change !

Because it's the main feature of Gear, here is a (little) list of things you can easily customize. Basically, the Sass functions of Gear have default parameters, and/or default values. By changing these values you take control of the css output.

  • Prefix

    You can prefix your gear classes. Which means, you can use it along another framework without conflicts. Gear will automatically prefix everything without you noticing. Yeah, that's cool.

  • Buttons

    Buttons are a big part of a flat webdesign. If you don't like the default buttons, here is what you can do :

    • colors
    • border radius
    • border color
    • sizes
    • padding
    • hover
    • …
  • Css reset

    Included by default, you can choose between 3 different resets.

  • Fonts

    Include all your favorite fonts like you would in a classic css file.

  • ...And much more, switch to the doc to discover all Gear capabilities.
